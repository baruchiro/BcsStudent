# CLAUDE.md

Guidance for Claude Code when working in this repository.

## About

Personal blog by Baruch Odem ("To be a computer science student"), mainly about
programming, written in **Hebrew**. Built with [Next.js](https://nextjs.org/)
(App Router) + [Contentlayer](https://contentlayer.dev/), styled with Tailwind,
deployed on [Vercel](https://vercel.com/). Based on the
[Tailwind Nextjs Starter Blog](https://github.com/timlrx/tailwind-nextjs-starter-blog).

## Commands

- `yarn dev` — start the dev server
- `yarn build` — fetch video metadata, build the site, run postbuild (also
  regenerates `src/app/tag-data.json`)
- `yarn serve` — serve the production build
- `yarn lint` — ESLint with `--fix` over `src` and `scripts`

Package manager is **Yarn** (Berry, `yarn@3.6.1`); Node `>=20`.

## Content & RTL

The blog is primarily in Hebrew and follows RTL (Right-to-Left) conventions:

- Write content in Hebrew.
- Use the `DirectionWrapper` component for proper text-direction handling.
- Navigation arrows should point **left** (←) for "forward" actions.

## Tags

- Each item (blog post / project / community) should have **2–5 tags**.
- Tags are **lowercase**, use **hyphens** for spaces, and are in **English**.
- **Reuse** existing tags when possible; the current tag list and counts live in
  `src/app/tag-data.json` (regenerated by `yarn build`).
- Tags must be specific and relevant to the main topics of the content.

For choosing, normalizing, reviewing, or cleaning up tags, use the
**organize-tags** skill.

## Blog post frontmatter

Each post must include `title` and `date`. Optional fields: `tags`, `lastmod`,
`draft`, `summary`, `images`, `authors`, `layout`, `canonicalUrl`.

Posts support: KaTeX math display, citations and bibliography, MDX components,
syntax highlighting with line numbers, image optimization, multiple authors, and
nested routing.

## Skills

Specialized workflows live in `.claude/skills/` and are invoked automatically
when relevant:

- **organize-tags** — choose, normalize, review, and clean up tags across blog
  posts, projects, and communities.
- **brand-guidelines** — brand colors, voice, typography, and asset usage when
  editing UI, styles, or content.
- **n8n-masking** — mask sensitive data in n8n workflow JSON before sharing it
  publicly.
